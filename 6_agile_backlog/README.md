# Agile / Jira Artifacts

This folder shows how the analysis findings translate into a real product backlog — the kind of artifact a Business Analyst would create and maintain in Jira, Azure DevOps, or similar tools.

**What's here:**

| File | What it shows |
|---|---|
| `user_stories.md` | 9 user stories grouped into 4 epics, each traced to a real data finding. Includes priority, story points, and acceptance criteria. |
| `kanban_board.md` | A Sprint 1 snapshot — stories move from To Do → In Progress → In Review → Done, with sprint metrics. |

**Why this matters for a German BA role:**

German employers listing Agile / Jira in job descriptions want to see that you can:
- Write stories a developer can build from (clear acceptance criteria)
- Prioritise based on business impact (data-backed, not guessing)
- Track work through a sprint cycle (Kanban board with metrics)
- Trace every story back to a real analysis finding (traceability)

Every number in these files comes from the actual data analysis — nothing is fabricated.
